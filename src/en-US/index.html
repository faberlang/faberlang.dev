Faber is a package-oriented programming language with a Latin behavioral vocabulary and a small, regular grammar. Source lowers through the radix compiler to reviewable Rust and native binaries. Its defining architectural property is that meaning lives in the semantic core, not in any rendering — neither the Latin you read nor the Rust it emits is the real program. Both are renderings of the same intermediate representation.

The name means maker or craftsman. Its compiler is named radix — root.

1. Semantic core

Structural meaning is carried by glyphs that do not belong to any human language: ← binds, → declares the success type, ⇥ declares the error channel, ≡ tests equality, ∪ unions types, and ∴ opens a compact branch. Behavioral vocabulary — functio, si, redde, fixum — is bindable. The two never trade places.

That split is what makes the rest of this article possible. Because structure is glyphic and only the words are bound, a rendering can change every word on the page without touching a single semantic fact.

  • Type-first declarations. functio f(numerus x), not f(x: numerus). Word order that happens to align with Chinese, Hindi, and Arabic.
  • Mechanical over magical. Nothing is inferred that a reviewer would have to guess.
  • The HIR is the truth. Translation always goes rendering A → HIR → rendering B. There is no direct path between two surfaces.

3. Compilation lanes

One frontend, one analysis pipeline, then a fork. The application lane emits from typed HIR — language-shaped output with declarations, annotations, and pretty control flow. The systems lane lowers HIR to MIR first — an execution-shaped control-flow graph — and emits from there.

Source → Lex → Parse → Collect → Resolve → Lower → Typecheck → Analysis │ ┌─────────────────────────┴─────────────────────────┐ │ │ HIR backends MIR backends │ │ Rust · Faber · TypeScript · Go fmir · wasm · llvm-text · metal-text · wgsl-text · sexp (+ CLI IR attachment for Rust)

The distinction is not cosmetic. HIR carries language shape — @ verte stdlib bindings, CLI programs, genus declarations — which is why Rust, Go, and TypeScript emit from it. MIR discards that shape in favour of a validated CFG with explicit runtime calls, which is what a GPU kernel or a Wasm module actually needs. A function can be routed explicitly with @ radix lane "mir" or "hir-direct"; unsupported lane/target combinations reject with a diagnostic rather than being silently ignored.

Systems targets are honest, not aspirational

The systems lane emits real artifacts for a fail-closed subset. Metal and WGSL emit compute-shader source for the device-safe kernel subset — rank-1 f32 device views are supported; rank-2 views reject. LLVM text emits .ll for external toolchain steps (NVVM → PTX downstream), not in-tree native codegen. None of these claim GPU launch or package delivery, and the CLI says so: run=no, package=no.

Rust is the behavioral reference. When a systems target can name a tensor runtime symbol but not execute tensor semantics, that is recorded as import/probe-only — the target can print the shape, not run it. Probe-only is not execution, and the matrix never lets one masquerade as the other.

4. Reader locale — the upward branch

Large language models have localized the conversation around programming and not the durable artifact. A Thai computer scientist can ask a model for help in Thai and still receive code whose control flow, APIs, and compiler errors are English-shaped. English proficiency becomes a gate to computer science rather than merely to conversation.

Reader locale is the response: the language a human uses to understand Faber — source, diagnostics, and optionally stdlib spellings — without English as a prerequisite. It is not application internationalization. It is opt-in, partial reader dialects over a semantic core that does not fork.

The loop

Human (locale) ──talk──► LLM (locale, pack-injected) │ ▼ Faber source (locale surface + glyphs) │ ◄──read/review── Human (locale) │ faber check │ (fail) ▼ Diagnostics (locale templates + codes) │ ◄──read/fix loop── Human + LLM │ ▼ Same HIR / same Rust binary as Latin twin

Why this is not AppleScript

Natural-language programming failed before because AppleScript and COBOL baked language shape into semantics — the localized surface was the program, and could not be escaped. Faber's symmetry guarantees no surface is privileged: any locale surface can become any other, including Latin, at any time. The localized surface is never a trap because it is never the only form the code can take.

5. Model-facing rendering — the inward branch

The symmetry predicts a third axis. Humans read Latin or Thai; machines execute Rust or Wasm; but an LLM is a distinct consumer class — it generates one token at a time, left to right, and cannot scan back to re-read what it wrote. A surface optimized for human reading is not necessarily optimal for model generation.

Token classVerdictReason
Word keywordsrewriteVerbose and not training-fluent. Drift to fn/def is the dominant emission-fidelity failure.
Structural glyphskeepAlready single-token and unambiguous. Their rarity in training makes them attention anchors that snap the model into Faber register and resist drift. The human-readable choice is also the generation-robust one.
BracesmeasureThe contested boundary. Braces maximize fluency but expose bracket-loss; named closers eliminate it but cost verbosity. No architectural answer — only measurement.
 The framework predicts that a model-facing rendering exists. It does not determine what it is; that is a per-model empirical question.

8. Status

Faber 1.0.0 freezes the grammar for the major line. The compiler, radix, is private; its output is not — the grammar, runtime, examples, tree-sitter package, and every lowered artifact are public and reviewable. The staging language exists so that the thing you review is the thing that runs.

Reader locale is proposed. It is a north-star design with real substrate prerequisites and an open pack-delivery architecture. This page renders illustrative pack spellings taken from the design document; no pack has been authored. Where the document defines no spelling, the token falls back to Latin and is marked and counted rather than invented — because vocabulary governance is a named risk, and the first pack's choices become the de-facto standard.
