*ثلاثة إشارات تصميمية تجعل شفرة Faber المصدرية مميزة من النظرة الأولى.*

تعتمد Faber ثلاث إشارات تصميمية متعمدة تعمل معًا لإنتاج شفرة مصدرية ذات هيئة نحوية مستقرة. يستطيع القارئ أن يتبين الدور الدلالي لكل بناء قبل أن يعرف إلى أي واجهة خلفية مستهدفة ستُصرَّف الشفرة.

## الإشارات الثلاث {#three}

| الإشارة | أمثلة | الدور |
|---------|-------|-------|
| تصريحات النوع أولاً | `نص nomen`، `عدد aetas` | هيئة النص تتجه نحو الربط — النوع، ثم الاسم. |
| كلمات لاتينية سلوكية | `functio`، `genus`، `si`، `redde`، `fixum` | تصريحات وعبارات ودورة حياة وقصد سلوكي. |
| رموز هيكلية | `← → ∴ ≡ ∪ ⇥` | تدفق القيم وتدفق الأنواع ووصلات هيكلية — عالمية لا تُترجم أبدًا. |

صُممت هذه الإشارات الثلاث بحيث تعزز بعضها بعضًا. القارئ الذي يعرف Faber في لغة محلية واحدة يستطيع قراءتها في أي لغة محلية أخرى لأن الرموز والهيكل لا يتغيران أبدًا. القارئ الذي يعرف واجهة Rust الخلفية يستطيع مع ذلك التعرف على شفرة Faber المصدرية لأن الكلمات اللاتينية المفتاحية وترتيب النوع أولاً ينتجان بصمة بصرية مميزة.

## تصريحات النوع أولاً {#type-first}

تضع Faber النوع قبل الاسم في كل تصريح. هذا عكس الصياغة السائدة في عائلة C، وهو مقصود:

| البناء | العادة في عائلة C | Faber |
|--------|-------------------|-------|
| متغير | `int count = 0` | `عدد count ← 0` |
| دالة | `fn greet(name: String) → String` | `دالة salve(نص nomen) → نص` |
| وسيط | `(String name)` | `(نص nomen)` |

تصريحات النوع أولاً تعني أن هيئة المعطى هي أول ما يراه القارئ. هذا يتوافق طبيعيًا مع اللغات التي تُقرأ من اليسار إلى اليمين للمدى الدلالي — التصريحات في الصينية والهندية والعربية تتبع الترتيب نفسه.

```faber
functio divide(numerus a, numerus b) → numerus ∪ nihil {
    si b ≡ 0 ergo redde nihil
    redde a / b
}
```

## المفردات السلوكية اللاتينية {#latin}

تستخدم Faber كلمات لاتينية لكل بناء له هيئة سلوكية أو نحوية. المفردات صغيرة ومنتظمة، مستمدة من مصدر كلاسيكي واحد بدلاً من الأصول الاشتقاقية المختلطة لمعظم لغات البرمجة.

### التصريحات {#declarations}

| الكلمة المفتاحية | الدور | المكافئ التقريبي |
|-------------------|-------|-------------------|
| `functio` | تصرح بدالة أو طريقة مسماة | `fn`، `def`، `function` |
| `genus` | تصرح بنوع ملموس ذي حقول | `class`، `struct` |
| `implendum` | تصرح بعقد سلوكي | `interface`، `trait` |
| `typus` | تصرح باسم مستعار لنوع | `typedef`، `type` |
| `discretio` | تصرح باتحاد موسوم | `enum`، `sum type` |

### الروابط والنقل {#bindings-and-transfer}

| الكلمة المفتاحية | الدور | المكافئ التقريبي |
|-------------------|-------|-------------------|
| `fixum` | ربط غير قابل للتغيير (يُكتب مرة واحدة) | `let`، `const` |
| `varia` | ربط قابل للتغيير | `let mut`، `var` |
| `sit` | ربط موجز غير قابل للتغيير مع استنتاج النوع | `let` (مستنتج) |
| `redde` | إرجاع قيمة من دالة | `return` |
| `iace` | رمي على قناة الخطأ | `throw`، `raise` |
| `mori` | مؤجل — سلوك لا يمكن التعبير عنه بعد | `unimplemented!`، `todo` |

### تدفق التحكم {#control-flow}

| الكلمة المفتاحية | الدور | المكافئ التقريبي |
|-------------------|-------|-------------------|
| `si` | تفرع شرطي | `if` |
| `sin` | تفرع وإلا-إذا | `else if` |
| `secus` | تفرع وإلا | `else` |
| `dum` | حلقة طالما | `while` |
| `itera` | تكرار (قيم أو مفاتيح أو مدى) | `for` |
| `elige` | مطابقة أنماط (الذراع الأولى تفوز) | `match`، `switch` |
| `fac` | كتلة محاولة مع استرداد من الخطأ | `try`، `do` |
| `cape` | معالج خطأ لـ fac | `catch` |

> المفردات اللاتينية **قابلة للربط** — تُشحن مع الحزمة المعيارية لكن يمكن إعادة تعيينها من خلال لغة القارئ المحلية. مبرمج تايلندي يرى `ถ้า` بدلاً من `si`؛ ومبرمج صيني يرى `函数` بدلاً من `functio`. المفردات ليست مميزة؛ القواعد النحوية وحدها هي المميزة.

## الرموز الهيكلية {#glyphs}

حيث تستخدم المفردات السلوكية كلمات لاتينية، يستخدم المعنى الهيكلي رموزًا عالمية. هذه الرموز لا تُترجم أبدًا ولا يتغير معناها أبدًا عبر طرق العرض. إنها المرساة البصرية التي تجعل شفرة Faber المصدرية مميزة بصرف النظر عن اللغة البشرية التي تُعرض بها الكلمات المفتاحية.

### تدفق القيم {#value-flow}

| الرمز | المعنى |
|-------|--------|
| `←` | ربط وقت التنفيذ وإعادة الإسناد والتغيير — عامل الإسناد الوحيد |
| `→` | تصريح نوع الإرجاع للدالة |
| `⇥` | مخرج بديل: نوع قناة الخطأ أو استرداد تحويل ضمني |
| `∴` | جسم موجز لذلك — يُدخل جسم عبارة وحيدة لتفرع |

### هيئة النوع {#type-shape}

| الرمز | المعنى |
|-------|--------|
| `∷` | نسبة نوع ساكنة — تأكيد وقت التصريف حول نوع قيمة |
| `↦` | تحويل وقت التنفيذ — تحليل أو إكراه قد يفشل |
| `∪` | نوع اتحاد ضمني — يصل بين نوعين (كما في `T ∪ لا شيء`) |

### المقارنة والمنطق {#comparison-and-logic}

| الرمز | المعنى |
|-------|--------|
| `≡` `≠` | تطابق تام وعدم تطابق تام — يتطلب تطابقًا صارمًا للأنواع |
| `<` `>` `≤` `≥` | مقارنات الترتيب |
| `∧` `∨` `⊻` `¬` | المنطقي والبتّي: و، أو، أو حصري، ليس |

### اتفاقية الربط مهمة {#the-binding-convention-matters}

اختيار رمز واحد يستحق انتباهًا خاصًا لأنه أكثر نقاط الالتباس شيوعًا للقراء الجدد:

| الرمز | الدور | الاستخدام لـ |
|-------|-------|-------------|
| `←` | **تدفق وقت التنفيذ** | الربط الأولي وإعادة الإسناد والتغيير في وقت التنفيذ |
| `=` | **الهيئة الهيكلية** | أسماء الحقول داخل النصوص الحرفية وبيانات التصريح الوصفية — وليس مخازن وقت التنفيذ |

معظم اللغات تفرط في تحميل `=` لكل من "عرّف هذا الحقل في نوع" و"ضع قيمة وقت تنفيذ في هذا المتغير." تفصل Faber بين هاتين المهمتين. كل `←` هو تدفق معطيات حي؛ وكل `=` داخل `Type { … }` هو تخطيط حقل جنس.

```text
# Runtime binding: ← attaches a value to a name
fixum numerus count ← 0
varia textus label ← "ready"
count ← count + 1

# Structural shape: = defines field values inside a literal
fixum _ p ← Point {
    x = 10,
    y = 20
}
```

## مقارنة باللغات السائدة {#compare}

يوضح الجدول أدناه كيف تناظر أنماط لغات البرمجة الشائعة نظام Faber ثلاثي الإشارات. يستخدم عمود Faber رمزًا مختلفًا أو كلمة مفتاحية مختلفة لكل مهمة دلالية مميزة — لا يوجد تحميل زائد.

| المهمة الدلالية | شائعة في اللغات الأخرى | Faber |
|-----------------|------------------------|-------|
| تصريح نوع الوسيط | `name: String` | `نص nomen` |
| نوع الإرجاع | `→ String`، `: String` | `→` `نص` |
| إسناد وقت التنفيذ | `x = value` | `←` |
| اختبار التطابق | `==` | `≡` |
| القابلية للعدم | `T?`، `Option<T>` | `T ∪ لا شيء` |
| تفرع + عبارة واحدة | `if (cond) return x` | `إذا cond ergo أرجع x` |
| تحويل النوع | `(T)value`، `value as T` | `value ∷ T` |
| تحويل (قد يفشل) | `try_into()` | `value ↦ T` |
