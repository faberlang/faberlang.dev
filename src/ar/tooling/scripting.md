بجانب مسار ⁦`rustc`⁩ المترجم، يدعم Faber التنفيذ التفسيري داخل العملية عبر مُنفّذ MIR.

## الاستخدام {#usage}

```bash
faber run --interpret script.fab
```

يشغّل هذا شيفرة Faber المصدرية داخل العملية بعد النصف الأمامي العادي من المترجم (التحليل حتى فحص الأنواع ثم تخفيض MIR)، دون استدعاء ⁦`rustc`⁩ أو إنشاء عملية بناء.

## آلية العمل {#how-it-works}

ينتج المترجم HIR محلّلاً وMIR محقّقاً وجدولاً محلولاً لدوال وقت التشغيل الجوهرية. يرسل منفّذ MIR كتل MIR مباشرة إلى مضيف، متجاوزاً دورة إصدار wasm وتهيئته:

```
Source → Lex → Parse → Collect → Resolve → Lower → Typecheck
                                                      ↓
                                                 MIR lowering
                                                      ↓
                                              MIR stepper + Host
```

## زمن الاستجابة {#latency}

يشغّل مسار البرمجة النصية نفس الواجهة الأمامية الخطية التي يستخدمها المسار المترجم، مضافاً إليها زمن المنفّذ المتناسب مع ما تنفّذه الشيفرة فعلياً:

| المرحلة | التكلفة |
|-------|------|
| الواجهة الأمامية (شيفرة من 100 سطر) | ~0.6 ملي ثانية |
| تنفيذ MIR | متناسب مع الجمل المنفّذة |

لا يستدعي المنفّذ ⁦`rustc`⁩ أبداً ولا ينشئ عملية، لذا فإن بدء التشغيل سريع بما يكفي ليشبه شيفرة shell.

## القيود {#limitations}

- لا يدعم منفّذ MIR جميع مسارات الإدخال/الإخراج للمضيف التي يدعمها المسار المترجم — تبقى بعض مغلّفات ⁦`norma:*`⁩ حصرية للمسار المترجم
- المنفّذ هو مشغّل تشخيصي ومرجعي أصلي لـ MIR، وليس بيئة تشغيل إنتاجية للتطبيقات المنشورة
- يبقى تجميع الحزم عبر Cargo هو مسار المنتج الأساسي
