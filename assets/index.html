Evaluate

Faber is for builders who want application code to stay readable when it crosses runtime and deployment boundaries. The honest RC1-era story is native applications, self-contained package binaries, and a narrow Go CLI lane; systems and accelerated lanes are tracked, not promised.

functio greet(name: Text) -> Text {

Learn

The first learning packet focuses on public-safe language concepts: intent-first source, the Latin keyword vocabulary, versioned contracts, diagnostics, and target capability labels. It avoids compiler implementation details and private proof vocabulary.

Capability Snapshot

  • Native application lane: evidenced locally; a release-profile CLI build reports faber 1.0.0-rc.1, while public run proof still requires release/export gates.
  • Package binary lane: evidenced locally through Cista package-store work, live registry remains gated.
  • Go CLI lane: narrow and experimental; do not describe it as a supported target.
  • GPU, browser, and broader backend lanes: tracked work, not RC1 product claims.

Release Gates

Before public use, this packet still needs a public/private export gate, release artifact or operator-approved install route, selected public examples with run evidence, link scrub, public artifact checksums, and operator approval for deployment or announcement.
