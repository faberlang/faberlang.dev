# Learn Faber

Version: `__DOCS_VERSION__`.

Status: local learning draft. Public use requires the Stage 1 export and
leakage gate.

## Learning Path

1. Read Faber source as application intent first.
2. Learn the keyword vocabulary.
3. Check syntax against the versioned grammar contract.
4. Use diagnostics to repair source.
5. Read target capability labels before assuming runtime behavior.

## Vocabulary

Faber leans into a distinctive Latin-derived vocabulary. The public learning
surface should teach that vocabulary directly and avoid implementation labels.

| Term | Draft meaning |
| --- | --- |
| `functio` | function declaration |
| `fixum` | fixed binding |
| `valor` | value |
| `ad` | effect route |
| `itera` | iteration |
| `mori` | fail or stop path |

The exact spelling set must come from the approved public language export
before this page is published.

## Learning Constraints

- Do not use private implementation terms as user-facing concepts.
- Do not imply every emitted backend is a runnable target.
- Do not cite private source paths or regression fixtures.
- Do not present local examples as public runnable examples until checked.
