*สัญญาณสามประการที่ทำให้มองเห็นได้ทันทีว่าแหล่งที่มาคือ Faber*

Faber เลือกใช้สัญญาณสามประการอย่างตั้งใจ และทั้งสามทำงานร่วมกันเพื่อสร้างแหล่ง
โค้ดที่มีรูปแบบไวยากรณ์คงที่ ผู้อ่านสามารถมองเห็นบทบาทเชิงความหมายของแต่ละ
โครงสร้างได้ก่อนจะรู้ด้วยซ้ำว่าโค้ดนั้นจะถูกคอมไพล์ไปยังแบ็กเอนด์เป้าหมายใด

## สัญญาณสามประการ {#three}

| สัญญาณ | ตัวอย่าง | บทบาท |
|--------|----------|------|
| การประกาศที่วางชนิดข้อมูลก่อน | `textus nomen`, `numerus aetas` | รูปแบบอ่านไปสู่การผูกค่า — ชนิดข้อมูล แล้วจึงชื่อ |
| คำพฤติกรรมภาษาละติน | `functio`, `genus`, `si`, `redde`, `fixum` | การประกาศ คำสั่ง วงจรชีวิต และเจตนาเชิงพฤติกรรม |
| อักขระโครงสร้าง | `← → ∴ ≡ ∪ ⇥` | การไหลของค่า การไหลของชนิดข้อมูล และจุดเชื่อมเชิงโครงสร้าง — เป็นสากลและไม่แปล |

ทั้งสามประการนี้ออกแบบมาให้เสริมกัน ผู้อ่านที่รู้จัก Faber ในภาษาหนึ่ง
สามารถอ่านได้ในทุกภาษาโลแคล เพราะอักขระและโครงสร้างไม่เคยเปลี่ยน
ผู้อ่านที่รู้จักแบ็กเอนด์ Rust ก็ยังจดจำแหล่งที่มาของ Faber ได้ เพราะคำสำคัญ
ภาษาละตินและลำดับชนิดข้อมูลก่อนชื่อสร้างรูปแบบการมองเห็นที่มีเอกลักษณ์

## การประกาศที่วางชนิดข้อมูลก่อน {#type-first}

Faber วางชนิดข้อมูลไว้ก่อนชื่อในการประกาศทุกชนิด ซึ่งตรงข้ามกับไวยากรณ์
ของภาษาตระกูล C ที่ใช้กันทั่วไป และเป็นการออกแบบโดยเจตนา:

| โครงสร้าง | รูปแบบที่คุ้นเคยในภาษาตระกูล C | Faber |
|-----------|----------------|-------|
| ตัวแปร | `int count = 0` | `numerus count ← 0` |
| ฟังก์ชัน | `fn greet(name: String) → String` | `functio salve(textus nomen) → textus` |
| พารามิเตอร์ | `(String name)` | `(textus nomen)` |

การประกาศที่วางชนิดข้อมูลก่อนทำให้โครงสร้างของข้อมูลเป็นสิ่งแรกที่ผู้อ่านเห็น
แนวทางนี้สอดคล้องตามธรรมชาติกับภาษาที่อ่านจากซ้ายไปขวาเพื่อขยายความหมาย
การประกาศในภาษาจีน ฮินดี และอาหรับก็ใช้ลำดับเดียวกัน

## คำศัพท์พฤติกรรมภาษาละติน {#latin}

Faber ใช้คำภาษาละตินกับโครงสร้างทุกชนิดที่มีรูปแบบเชิงพฤติกรรมหรือไวยากรณ์
คำศัพท์มีจำนวนน้อยและเป็นระบบ โดยมาจากแหล่งภาษาคลาสสิกเดียว แทนที่จะ
ใช้รากศัพท์หลากหลายปะปนกันเหมือนในภาษาการเขียนโปรแกรมส่วนใหญ่

### การประกาศ {#declarations}

| คำสำคัญ | บทบาท | คำเทียบเคียงโดยประมาณ |
|---------|------|------------------------|
| `functio` | ประกาศฟังก์ชันหรือเมธอดที่มีชื่อ | `fn`, `def`, `function` |
| `genus` | ประกาศชนิดข้อมูลรูปธรรมที่มีฟิลด์ | `class`, `struct` |
| `implendum` | ประกาศสัญญาเชิงพฤติกรรม | `interface`, `trait` |
| `typus` | ประกาศนามแฝงของชนิดข้อมูล | `typedef`, `type` |
| `discretio` | ประกาศยูเนียนแบบมีแท็ก | `enum`, `sum type` |

### การผูกค่าและการส่งผ่าน {#bindings-and-transfer}

| คำสำคัญ | บทบาท | คำเทียบเคียงโดยประมาณ |
|---------|------|------------------------|
| `fixum` | การผูกค่าแบบเปลี่ยนแปลงไม่ได้ (เขียนได้ครั้งเดียว) | `let`, `const` |
| `varia` | การผูกค่าแบบเปลี่ยนแปลงได้ | `let mut`, `var` |
| `sit` | การผูกค่าแบบเปลี่ยนแปลงไม่ได้ที่อนุมานชนิดข้อมูลอย่างกระชับ | `let` (อนุมานชนิดข้อมูล) |
| `redde` | คืนค่าจากฟังก์ชัน | `return` |
| `iace` | โยนข้อผิดพลาดผ่านช่องทางข้อผิดพลาด | `throw`, `raise` |
| `mori` | เลื่อนการทำงาน — ยังไม่สามารถแสดงพฤติกรรมนี้ได้ | `unimplemented!`, `todo` |

### การควบคุมการไหล {#control-flow}

| คำสำคัญ | บทบาท | คำเทียบเคียงโดยประมาณ |
|---------|------|------------------------|
| `si` | แขนงเงื่อนไข | `if` |
| `sin` | แขนงเงื่อนไขอื่น | `else if` |
| `secus` | แขนงอื่น | `else` |
| `dum` | วนซ้ำตราบเท่าที่เงื่อนไขเป็นจริง | `while` |
| `itera` | การวนซ้ำ (ค่า คีย์ หรือช่วง) | `for` |
| `elige` | จับคู่รูปแบบ (เลือกแขนงแรกที่ตรงกัน) | `match`, `switch` |
| `fac` | บล็อกลองทำงานพร้อมกู้คืนจากข้อผิดพลาด | `try`, `do` |
| `cape` | ตัวจัดการข้อผิดพลาดสำหรับ `fac` | `catch` |

> คำศัพท์ภาษาละตินนี้ **ผูกแทนได้** — มาพร้อมแพ็กมาตรฐาน แต่สามารถแมปใหม่
> ผ่านโลแคลของผู้อ่านได้ โปรแกรมเมอร์ไทยเห็น `ถ้า` แทน `si`;
> โปรแกรมเมอร์จีนเห็น `函数` แทน `functio` คำศัพท์ไม่ได้มีสิทธิพิเศษ
> มีเพียงไวยากรณ์เท่านั้นที่คงเดิม

## อักขระโครงสร้าง {#glyphs}

ขณะที่คำศัพท์เชิงพฤติกรรมใช้คำภาษาละติน ความหมายเชิงโครงสร้างใช้อักขระสากล
อักขระเหล่านี้ไม่เคยถูกแปลและไม่เคยเปลี่ยนความหมายระหว่างการเรนเดอร์
อักขระเหล่านี้เป็นจุดยึดทางสายตาที่ทำให้จดจำแหล่งที่มาของ Faber ได้
ไม่ว่าคำสำคัญจะถูกเรนเดอร์ด้วยภาษามนุษย์ใด

### การไหลของค่า {#value-flow}

| อักขระ | ความหมาย |
|-------|---------|
| `←` | การผูกค่าขณะรันไทม์ การกำหนดค่าใหม่ และการกลายค่า — เป็นตัวดำเนินการกำหนดค่าเพียงตัวเดียว |
| `→` | การประกาศชนิดค่าที่ฟังก์ชันคืนกลับ |
| `⇥` | ทางออกสำรอง: ชนิดข้อมูลของช่องทางข้อผิดพลาด หรือการกู้คืนจากการแปลงค่าแบบอินไลน์ |
| `∴` | บอดีแบบ therefore อย่างกระชับ — นำเข้าบอดีของแขนงที่มีคำสั่งเดียว |

### รูปแบบชนิดข้อมูล {#type-shape}

| อักขระ | ความหมาย |
|-------|---------|
| `∷` | การระบุชนิดข้อมูลแบบสแตติก — การยืนยันชนิดข้อมูลของค่าในเวลาคอมไพล์ |
| `↦` | การแปลงค่าขณะรันไทม์ — การแยกวิเคราะห์หรือบังคับชนิดข้อมูลที่อาจล้มเหลว |
| `∪` | ชนิดยูเนียนแบบอินไลน์ — เชื่อมชนิดข้อมูลสองชนิดเข้าด้วยกัน (เช่น `T ∪ nihil`) |

### หลักการผูกค่ามีความสำคัญ {#the-binding-convention-matters}

การเลือกใช้อักขระหนึ่งควรได้รับความสนใจเป็นพิเศษ เพราะเป็นจุดที่ผู้อ่านใหม่
มักสับสนมากที่สุด:

| อักขระ | บทบาท | ใช้สำหรับ |
|-------|------|---------|
| `←` | **การไหลขณะรันไทม์** | การผูกค่าเริ่มต้น การกำหนดค่าใหม่ และการกลายค่าขณะทำงาน |
| `=` | **รูปแบบเชิงโครงสร้าง** | ชื่อฟิลด์ภายในลิเทอรัลและข้อมูลเมทาดาทาของการประกาศ — ไม่ใช่การจัดเก็บค่าขณะรันไทม์ |

ภาษาส่วนใหญ่ใช้ `=` ซ้ำกันสำหรับทั้ง “กำหนดฟิลด์นี้ในชนิดข้อมูล”
และ “ใส่ค่าขณะรันไทม์ลงในตัวแปรนี้” Faber แยกหน้าที่ทั้งสองออกจากกัน
ทุก `←` คือการไหลของข้อมูลที่ทำงานอยู่ ส่วนทุก `=` ภายใน `Type { … }`
คือเค้าโครงฟิลด์ของ `genus`

## เมื่อเทียบกับภาษาที่ใช้กันทั่วไป {#compare}

ตารางด้านล่างแสดงวิธีที่รูปแบบการเขียนโปรแกรมทั่วไปแมปเข้ากับระบบ
สัญญาณสามประการของ Faber คอลัมน์ Faber ใช้อักขระหรือคำสำคัญที่แตกต่างกัน
สำหรับงานเชิงความหมายแต่ละประเภท — ไม่มีการใช้ซ้ำหลายความหมาย

| งานเชิงความหมาย | รูปแบบที่พบในภาษาอื่น | Faber |
|--------------|---------------------------|-------|
| การประกาศชนิดข้อมูลของพารามิเตอร์ | `name: String` | `textus nomen` |
| ชนิดค่าที่คืนกลับ | `→ String`, `: String` | `→` `textus` |
| การกำหนดค่าขณะรันไทม์ | `x = value` | `←` |
| การทดสอบความเท่ากัน | `==` | `≡` |
| ค่าที่อาจว่าง | `T?`, `Option<T>` | `T ∪ nihil` |
| แขนงพร้อมคำสั่งเดียว | `if (cond) return x` | `si cond ∴ redde x` |
| การแคสต์ชนิดข้อมูล | `(T)value`, `value as T` | `value ∷ T` |
| การแปลงค่าที่อาจล้มเหลว | `try_into()` | `value ↦ T` |
