## วิธีการทำงาน {#how-it-works}

คอมไพเลอร์จะสร้าง HIR ที่ผ่านการวิเคราะห์แล้ว MIR ที่ผ่านการตรวจสอบ และตาราง runtime intrinsic ที่แก้ความกำกวมแล้ว MIR stepper จะส่งต่อบล็อก MIR ไปยังโฮสต์โดยตรง และข้ามรอบการสร้างและสร้างอินสแตนซ์ wasm:

## เวลาแฝง {#latency}

เส้นทางสคริปต์ใช้ส่วนหน้าเชิงเส้นเดียวกับเส้นทางที่คอมไพล์แล้ว และเพิ่มเวลาสำหรับ stepper ตามสัดส่วนของสิ่งที่สคริปต์เรียกใช้งานจริง:

| ระยะ | ค่าใช้จ่าย |
|-------|------|
| ส่วนหน้า (สคริปต์ 100 บรรทัด) | ~0.6 ms |
| การประมวลผล MIR ทีละขั้น | แปรผันตามจำนวนคำสั่งที่ทำงาน |

stepper จะไม่เรียกใช้ `rustc` หรือเริ่มกระบวนการใด ๆ ดังนั้นเวลาเริ่มต้นจึงเร็วพอที่จะให้ความรู้สึกเหมือนสคริปต์ shell

## ข้อจำกัด {#limitations}

- MIR stepper ไม่รองรับเส้นทาง I/O ของโฮสต์ทั้งหมดที่เส้นทางคอมไพล์แล้วรองรับ — wrapper บางส่วนของ `norma:*` ยังใช้งานได้เฉพาะในเส้นทางคอมไพล์
- stepper เป็นตัวประมวลผลสำหรับการวินิจฉัยและการอ้างอิงที่ทำงานบน MIR โดยตรง ไม่ใช่ runtime สำหรับใช้งานจริงในแอปพลิเคชันที่นำไป deploy
- การคอมไพล์แพ็กเกจผ่าน Cargo ยังคงเป็นเส้นทางผลิตภัณฑ์หลัก
